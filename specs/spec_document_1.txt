- 1 Gold Chest per level

Combat Levels 11-20:
- 250,000 Credux per level
- 1 Boss Treasure Chest per level

Combat Levels 21-30:
- 500,000 Credux per level
- 2 Boss Treasure Chests per level

Combat Levels 31-40:
- 1,000,000 Credux per level
- 3 Boss Treasure Chests per level

Combat Levels 41-50:
- 5,000,000 Credux per level
- 1 Boss Golden Treasure Chest per level

Requirements:

- Rewards must be granted for each individual level gained.
- Support gaining multiple levels from one action.
- Calculate rewards separately for every crossed reward bracket.
- Do not grant rewards based only on the final level.

Examples:

- Level 10 to 11 grants the Level 11 reward.
- Level 19 to 22 grants rewards for Levels 20, 21, and 22.
- Level 39 to 41 grants the Level 40 reward and the Level 41 reward.

The level-up response should show:

- Previous Combat Level
- New Combat Level
- Total Credux received
- Chests received, grouped by chest type

4. RETROACTIVE COMPENSATION FOR ALL CURRENT USERS
==================================================

All existing users must receive the rewards they should have earned based on their current Combat Level and Believer Level.

Create a production-safe compensation script or migration process that checks every current user.

For each user:

1. Read their current Character Combat Level.
2. Read their current Believer Level.
3. Determine which level rewards have already been credited, if reward records exist.
4. Calculate all missing rewards from Level 1 up to their current level.
5. Credit only the missing rewards.
6. Store reward records so the same compensation cannot be granted twice.

If this is a completely new reward system and no previous level rewards exist, compensate every eligible level from Level 1 through the user's current level.

Requirements:

- The process must be idempotent.
- It must be safe to rerun.
- Include a dry-run mode.
- Dry-run mode must not change balances, inventories, or reward records.
- Process users in batches to avoid excessive memory usage or database load.
- Use transactions per user or per safe batch.
- A failure for one user must not corrupt another user's compensation.
- Log successful, skipped, and failed compensation records.
- Do not load all users and inventories into memory at once.

The compensation result must report:

- Total users checked
- Total users compensated
- Total users skipped
- Total failed users
- Total Credux distributed
- Total Gold Chests distributed
- Total Boss Treasure Chests distributed
- Total Boss Golden Treasure Chests distributed

Provide:

- The compensation script

- Production execution command
